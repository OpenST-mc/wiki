# OpenST档案馆新人须知：简介是怎么写的？

一个合格的简介应保持客观描述，仅陈述功能与结构信息。

以下内容不应出现：

- 主观评价（如“推荐使用”“更好的设计”）
- 面向读者的表达（如“你可以…”“我们…”）
- 与机器本身无关的背景或宣传性内容


### 机器概览应包含以下内容：

**必填：**

* 核心功能说明
* 使用版本

**建议填写：**

* 投影创建者名称（用于溯源）
* 投影创建时间

**可选：**

* 借物表（如存在依赖资源）
* 感谢名单


## 2. 使用说明又该填什么？
使用说明应覆盖完整操作流程，使未接触过该机器的用户能够独立运行。

建议包含：

1. 投影放置方式（方向 / 对齐方式）
2. 启动条件（红石 / 手动操作等）
3. 输入方式（如有）
4. 输出或结果说明

## 简介内容不得包含任何形式的组织，服务器或个人宣传信息。
